The Customer has ordered an instrument for analysis of urine samples. The system consists of multi axle arms, hydraulic pumps, and digital optics. Being a small software team we get to dip our noses in many different layers like React GUI, Linux Applications, Yocto BSP, bus communications, and low level software for in house developed electronics. My primary focus was och the low level software, applications, and some tools needed in the testing and development, largely in Python. April 2017 - April 2018
Prevas AB - Cargotec Sweden AB
HIAB constructs loader cranes and demountables for trucks. On the crane computers they are now putting a gateway that are to transfer statistics to a cloud service, for presentation and aggregation. My task was to develop the Linux application responsible for communications between product (crane, demountable) through data bus (today serial and CAN) and cloud service. Taking the data and packaging it in JSON format to be sent via MQTT. The team was extremely distributed with colleagues in Poland, Tampere, Söderhamn, and Hudiksvall. Other tightly coupled teams can be found in India, Linköping, and Norway. The software developed is based on a "proof of concept" software that piece by piece is rewritten and expanded. All new and rewritten code are to follow C++11. Hardware layers are to be abstracted to allow for larger parts of the application to be re-used as hardware and Linux platform are subjects of change.

Prevas AB - InMotion
InMotion is a company that develop and produce drives for electrical vehicles. My assignment was to create an automated regression test framework of a "drive" via CAN using requirement documents. Prevas AB - Interspiro
This is a project in which Prevas has developed a communications solution for firemens breathing apparatus. My first assignment in this project concerns reorganization or configuration switches. Instead of having separate source codes for different units we add dynamic switches who condition the execution based on a parameterized config file added in the memory. My work span investigating effects on the code and designing the conditional structure. The project is written in C and executed on an Ti 16-bit MSP430 processor.
July 2014 - August 2014
Prevas AB - Interspiro
I was given the task to compose and test a test environment that was to be used for test driven development. The environment was created in a virtual machine, Jenkins was used as CI tool, CUnit as test framework for the sample test cases also created and Make-files specifik for the C code written for MPS430 hardware was created. The project was documeted in a guide describing step by step how such an environment can be created in basically any Linux environment.
